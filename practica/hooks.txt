Hooks principios:
. Siempre tienen que ser invocados en el mismo orden (no poner mas abajo de condicionales)

_useState_
hook mas comun que permite actualizar un estado

_useEffect_
hook que se usa para manejar efectos secundarios/cosas no controladas por el usuario. Por ejemplo: si decidimos usar una funcion para hacer
fetch, y ponerla en un onClick, no necesitamos usar useEffect porque no es un efecto secundario, es un efecto controlado por el usuario y
ademas useEffect debe ser ejecutado durante el ciclo de vida de el componente, no cuando el usuario haga click/un evento.

Se ejecuta cuando se monta el componente, cuando hay un cambio en un state(porque se hace un render), cuando cambia la dependencia []

useEffect puede tener un return()={} que sirve como cleaner para los efectos secundarios. Se ejecuta antes de que el componente se desmonte
o renderize. Ejemplos de cuando se usa:
Limpiar timers o intervalos.
Cancelar peticiones de red.
Eliminar event listeners que fueron añadidos en el useEffect.

inputs: en las propiedad onChange tenemos que agregar una funcion que use el manejador de estado y ponga en el estado el valor de e.target.memoizedValue
        en el value tenemos que poner ese estado

_useCallback_
Memoriza funciones para evitar que se re-dibujen con cada re-renderizado, a menos que la dependencia que le asignamos cambie (dejando el array
vacio hace que no se repinta mas la funcion en re-renderizados posteriores)
util cuando le pasamos funciones complejas como props a componentes hijos, ya que cada vez que el componente padre se re-renderice, la prop 
que le pasamos al hijo se volvera a crear. O tambien cuando en nuestro componente hay una funcion compleja y no queremos que se re-dibuje
Cuando no usar? si la funcion no causa problemas de rendimiento o desencadenado de re-renderizados innecesarios.
Ya que el uso excesivo de esto puede hacer el codigo sucio y tambien consume recursos
ej:
const memoizedCallback = useCallback(() => {
  // creamos esta funcion memoizedCallback y no se volvera a crear en los proximos re-renderizados.
}, []);

_useMemo_
Memoriza resultados complejos de funciones o variables para que no se re-calculen en cada re-renderizado, a menos que cambien las dependencias.
Se usa mucho para memorizar cosas que no se tienen que volver a ejecutar por cosas externas a la funcion que da un resultado en si,
por eso la dependencia le dice cuando es que se tiene que ejcutar. Por ejemplo tener un hook aparte que causa un re-renderizado y por culpa
de esto se vuelve a hacer todo el calculo complejo de una funcion que no tuvo nada que ver con esta ultima.
ej:
const memoizedValue = useMemo(() => {
  return someExpensiveCalculation(); //retornamos el valor de esta funcion gracias a "return", memoizedValue tendra este resultado
}, []);

_useRef_
toma una referencia del dom (sin renderizar el componente) y guarda la informacion de su estado actual, es mutable y no causa una
re-renderizacion cuando cambia, osea que puede cambiar su valor y no disparar un re-renderizado
. la referencia no desaparece y sigue apuntando al mismo objetivo incluso cuando se re-renderiza el componente
. si la referencia en el re-renderizado desaparece, la ref sera null o el nuevo elemento
